# Multi-Terminal 产品 UI/UX 设计规范与页面设计文档

## 1. 设计愿景与总体风格 (Design Vision & Style)

本项目致力于为研发与运维工程师打造一款沉浸式、极客感且高度专业的多主机管理终端。由于目标用户需长时间盯着屏幕处理大量代码、日志和服务器状态，界面的**可用性 (Usability)**、**信息层级 (Information Hierarchy)** 和 **视觉舒适度 (Visual Comfort)** 是设计的核心。

### 1.1 核心设计原则
*   **极简且专业 (Minimal & Professional)**: 摒弃花哨的装饰，将 90% 的屏幕空间留给终端核心内容。
*   **深色模式优先 (Dark Mode First)**: 采用护眼的深色主题（Dark Theme），减少长时间注视带来的视觉疲劳。
*   **状态强感知 (Strong State Awareness)**: 通过色彩和动效，明确反馈当前机器的状态（健康、慢节点、TUI锁定、危险警告）。
*   **极致的交互效率 (Extreme Efficiency)**: 支持全键盘操作，减少鼠标点击。

### 1.2 全局色彩规范 (Color Palette)
*   **背景色 (Background)**:
    *   主背景 (Main Bg): `#121212` (深灰黑，降低刺眼感)
    *   侧边栏/面板背景 (Panel Bg): `#1E1E1E`
    *   悬停/选中状态 (Hover/Active): `#2D2D2D`
*   **文本色 (Text)**:
    *   主文本 (Primary Text): `#E0E0E0` (柔和白)
    *   次要文本 (Secondary Text): `#858585`
*   **状态语义色 (Semantic Colors)**:
    *   **健康/激活 (Active/Success)**: `#00C853` (极客绿) - 用于已连接、当前激活的终端边框。
    *   **警告/慢节点 (Warning/Slow)**: `#FFAB00` (琥珀黄) - 用于提示网络延迟、缓冲区积压的节点。
    *   **危险/锁定 (Danger/Blocked)**: `#FF3D00` (警示红) - 用于 TUI 模式锁定、命中高危风控规则的终端。
    *   **批量执行 (Multiexec)**: `#2979FF` (科技蓝) - 用于标识当前处于批量广播模式。

## 2. 核心页面结构布局 (Layout Structure)

整体采用经典的 **“上-左-中-右”** 布局，兼顾全局导航与沉浸式操作。

### 2.1 顶部导航栏 (Top Navbar)
*   **高度**: 48px，极其紧凑。
*   **左侧**: Logo & 产品名称 (Multi-Terminal)。
*   **中间**: 全局模式切换 Toggle（单机模式 Single Mode / 批量广播模式 Multiexec Mode）。
*   **右侧**: 当前登录用户信息、全局设置入口、快捷键帮助 (Keybinds) 按钮、系统状态指示灯（当前在线 Session 数、网关负载）。

### 2.2 左侧边栏 - 资产树 (Left Sidebar - Asset Tree)
*   **宽度**: 可收起，展开时约 240px。
*   **功能**:
    *   **顶部搜索框**: 支持模糊搜索主机名、IP、标签 (Tags)。
    *   **主机列表/树形结构**: 按项目或机房分组显示服务器。
    *   **状态图标**: 每个主机节点前有状态点（绿=在线，灰=离线）。
    *   **交互**:
        *   单机点击：在主区域打开该主机的终端。
        *   Checkbox 勾选：将该主机加入批量执行目标组。
        *   支持按组 (Group) 一键全选。

### 2.3 中间主操作区 - 聚合工作台 (Main Area - Unified Workspace)
为了避免“满屏黑框框”造成的视觉灾难和信息过载，主操作区在批量执行（Multiexec）模式下采用**“统一输入 + 智能输出折叠”**的创新设计，摒弃传统的终端网格平铺。

*   **单机模式 (Single Mode)**:
    *   全屏显示当前选中的单一主机终端（传统的黑框框）。
*   **批量模式 (Multiexec Mode) 核心布局**:
    *   **超级控制台 (Master Terminal)**: 占据屏幕上半部分或主要区域的单一终端黑框。用户只在这个唯一的框内输入命令，这代表了对所有选中主机的统一下发通道。
    *   **执行状态底座 (Execution Dock)**: 位于 Master Terminal 下方的状态面板，实时显示命令在各台机器上的执行进度。
        *   **进度环/进度条**: 显示“已完成 45/50，失败 2，执行中 3”。
        *   **微型状态卡片**: 每个目标主机抽象为一个带有状态灯（绿、黄、红、转圈动画）的小卡片，而非完整的黑框。
*   **输出智能聚合 (Smart Output Aggregation)**:
    *   **Diff 分组**: 当多台机器返回相同的结果时（如检查 Nginx 状态，48台机器均返回 `active (running)`），系统将这 48 台机器的输出合并为一条显示在 Master Terminal 中，并标注 `[48 Hosts identical output]`。
    *   **异常分离**: 针对输出不一致或执行失败的节点，系统在 Master Terminal 右侧或下方独立拉出“异常输出面板 (Exception Panel)”或“日志抽屉 (Log Drawer)”，让用户只关注出错的部分。

### 2.4 右侧边栏 - 审计与风控面板 (Right Sidebar - Audit & Info) (可选/可收起)
*   **宽度**: 300px，默认收起状态。
*   **功能**:
    *   **实时事件流**: 当命中风控规则（如执行了 `rm -rf`）时，此处实时滚动红色告警信息。
    *   **节点详情**: 选中某台主机时，显示该主机的 CPU/内存基本信息（若有监控集成）。
    *   **Multiexec 状态统计**: 批量执行时，统计成功执行、卡顿、失败的节点比例。

## 3. 关键场景界面设计 (Key User Flow Designs)

### 3.1 极速批量巡检 (Unified Multiexec Mode)
**设计目标**: 让用户明确感知“我正在对多台机器输入”，同时避免满屏相同输出造成的视觉噪音。
*   **统一直觉输入**:
    *   开启 Multiexec 后，界面平滑过渡，多个原本平铺的小终端收束合并为一个巨大的 **Master Terminal**。
    *   顶部明确标识：`[Broadcast Mode] Target: 50 Hosts`。
*   **动态输出面板 (Dynamic Feedback)**:
    *   **交互式等待**: 命令下发后，Master Terminal 内部暂不疯狂刷屏。下方出现细长的进度条或微卡片，展示 50 台机器的响应状态（🟢返回成功 / 🟡仍在执行 / 🔴报错拦截）。
    *   **聚合回显**: 系统自动执行 Diff，将大多数机器的相同标准输出折叠为一条回显；仅将存在差异的节点日志提取出来，在下方以手风琴折叠面板 (Accordion) 的形式高亮展示。
*   **智能屏蔽与中断**: 如果某台机器处于 `vim` 等 TUI 模式，它不会阻塞其他机器的执行。下方的微卡片中，该机器会被标为红色 🔒，提示“处于非 Shell 状态，已隔离”，用户可一键将这部分异常节点剔除出当前广播组。

### 3.2 慢节点视觉隔离 (Slow Node Isolation)
**设计目标**: 让卡顿节点显眼，但不阻断用户视线。
*   **触发表现**: 当某海外节点网络不佳，缓冲区积压超过 85% 时。
*   **视觉变化**: 该终端卡片的边框逐渐变为**琥珀黄 (Yellow)**。终端右上角出现一个警告图标 ⚠️，提示“输出积压，正在限流...”。
*   **恢复表现**: 当积压消化完毕，黄色边框平滑过渡回正常颜色，警告图标消失。

### 3.3 全息审计防抵赖 (Audit Replay Player)
这是一个独立的页面，类似于视频播放器。
*   **页面布局**: 居中的大型终端屏幕，底部是时间轴 (Timeline) 和播放控制条。
*   **控制条组件**:
    *   播放/暂停/停止按钮。
    *   倍速选择 (1x, 2x, 4x)。
    *   进度拖拽滑块 (Scrubber)。
*   **事件高亮**: 在底部的进度条上，用明显的标记（如红点）标出用户执行高危命令或发生错误的时间点，方便审计员一键跳转。
*   **元数据面板**: 在播放器右侧展示会话信息（操作人 ID、目标 IP、开始时间、结束时间、脱敏后的输入流记录）。

### 3.4 登录与无痛断线重连 (Login & Reconnect)
*   **登录页**: 极简深色背景，居中一个带毛玻璃效果 (Glassmorphism) 的登录框，支持 SSO (SSO Login) 和常规密码登录。
*   **断线重连 UI**: 当用户合上电脑休眠，网络断开时。
    *   整个界面变暗，中央弹出 Loading 动画，提示 “网络连接已断开，正在尝试恢复会话 (Reconnecting...)”。
    *   重连成功后，提示框消失，终端自动回放离线期间错过的日志，屏幕闪过一道绿色的“成功”提示光带。

## 4. 微交互与动效细节 (Micro-interactions)

*   **输入锁定 (Echo Lock)**: 在 Multiexec 模式下，为了防止回显重影，前端渲染会有极短暂的锁定。设计上，在按键按下的瞬间，字符以轻微的粗体/高亮态立即呈现在屏幕上，0.1秒后恢复正常字体，给用户一种“敲击极其顺滑且已经被确认”的爽快感。
*   **风控拦截弹窗 (Risk Intercept Modal)**:
    *   当输入中危命令（如 `reboot`）触发拦截时，屏幕中央弹出带震动动效的警告对话框。
    *   **设计**: 对话框背景深灰，文字红色。必须输入特定的确认词（如 "CONFIRM"）或倒计时 3 秒后才能点击“强制执行”按钮，防止误点。

## 5. 响应式与可用性保障 (Accessibility)
*   **字体选择**: 终端区域强制使用等宽编程字体（推荐 `Menlo`, `Monaco`, `Fira Code`, `JetBrains Mono`），支持连字 (Ligatures) 显示。
*   **快捷键支持**: 提供全局快捷键唤出 Cheat Sheet (`Cmd/Ctrl + /`)。
    *   `Cmd/Ctrl + Click`：多选主机。
    *   `Shift + Click`：连选主机。
    *   `Alt + M`：快速切换 Multiexec 模式。
*   **高对比度**: 确保所有文本与背景的对比度达到 WCAG AA 标准，即使在屏幕反光的环境下也能看清命令行输出。
