# 用户故事与业务场景

## 1. 产品愿景与目标
本系统致力于在现代浏览器中，为研发与运维工程师提供一站式、高性能的多主机实时终端能力。它不但要在视觉与体验上媲美原生客户端（如 Xshell、iTerm2），还需要在“多机协同执行”这一高频场景下，通过一系列创新机制实现绝对的安全可控、差异化识别以及事后审计追踪。

## 2. 目标用户画像
- **SRE / 运维专家**: 面对上百台业务实例，需要快速验证配置、排查批量网络问题。
- **安全审计员**: 需要无死角地复盘线上事故，查看谁在何时敲下了哪些命令，其界面反应如何。
- **后端研发工程师**: 日常快速登录跳板机查看少量日志，或者进入容器排查崩溃现场。

## 3. 核心用户故事 (User Stories)

### 3.1 极速批量巡检 (Multiexec)
> **作为** 一名运维工程师
> **我希望** 能够在界面上勾选 10 台 Web 服务器，并在当前焦点终端中只敲击一次 `tail -f /var/log/nginx/error.log`
> **以便于** 这 10 台机器能够同步接收到该指令并实时输出结果，让我能一目了然地看到整个集群的报错情况，免去了我依次复制粘贴的繁琐操作。

**验收标准 (AC):**
1. 可以在 UI 上多选多台机器。
2. 键盘输入字符时，被选中的终端能够实时且同步接收到相同按键。
3. 输入控制字符（如 `Ctrl+C`）时，能同步中止所有被选机器的任务。
4. **Edge Case (部分执行成功)**: 若在 Multiexec 广播执行期间，其中一台目标主机突然断网，其余 9 台机器应不受任何影响继续执行并输出。前端针对断网的机器予以置灰标红提示，但不阻塞用户的进一步输入操作。

### 3.2 智能防误触发 (上下文一致性)
> **作为** 一名经验不足的研发人员
> **我希望** 在使用批量广播命令前，系统能自动帮我检查这几台机器的状态
> **以便于** 如果我其中一台机器还不小心留在了 `vim` 编辑器里，系统能将其标红并拦截我对它的广播，防止我原本想批量敲的 `rm -rf /tmp/cache` 变成在 `vim` 里疯狂删除乱码，从而造成线上事故。

**验收标准 (AC):**
1. 广播前，系统隐蔽发送探针（如 ``）。
2. 将非 `shell` 状态的终端加上红色遮罩，并阻止对其广播写入。
3. 正常的 `shell` 节点不受影响，继续执行指令。

### 3.3 无痛断线重连 (会话挂起与恢复)
> **作为** 一个经常带电脑在公司各个会议室跑来跑去的人
> **我希望** 当我的笔记本合上盖子休眠，或者切换了 WiFi 导致网络断开时，远端的执行进程不要被杀掉
> **以便于** 当我 5 分钟后重新打开网页，能看到刚才正在编译的 `make build` 命令还在继续跑，而且丢失的输出已经被重新同步到了屏幕上。

**验收标准 (AC):**
1. WS 连接断开时，后台保持 SSH 通道活跃（进入 `detached` 状态），不发送 KILL 信号。
2. 在 TTL（10分钟）内重新连接时，前端能无缝恢复会话。
3. 离线期间的日志输出将被暂存在 Buffer 中，并在重连时一次性回放到前端屏幕。

### 3.4 慢节点隔离与视觉反馈
> **作为** 一名在弱网环境下管理海外节点的主管
> **我希望** 当我对美东、美西、欧洲三个机房同时执行大量输出查询时，如果欧洲节点卡顿
> **以便于** 欧洲节点的卡顿不会让浏览器卡死或者拖慢美东节点的输出速度，并且 UI 会明显提示我哪个节点掉队了，让我可以从容地中断其进程。

**验收标准 (AC):**
1. 当某个节点发生严重输出积压时，后端触发限流，标记节点为慢节点。
2. 前端展示视觉警告（如变黄）。
3. 慢节点的卡顿绝不影响其他正常节点的交互速度。
4. **Edge Case (恢复机制)**: 当慢节点的积压 Buffer 随着时间逐渐被前端消费完毕，水位回落到安全阈值后，后端自动解开反压，前端移除视觉警告，该节点无缝恢复到正常的实时同步状态，且期间无任何历史输出丢失。

### 3.5 100% 全息审计防抵赖
> **作为** 一名公司的安全合规专员
> **我希望** 所有的终端交互不仅仅记录文字命令，还能连同整个界面展现一并录像存储下来
> **以便于** 当出现“删库跑路”或者隐蔽破坏行为时，我可以像播放视频一样，完整还原当时攻击者的确切操作屏幕（包含控制字符的展现），提供不可伪造的司法级证据。

**验收标准 (AC):**
1. 将所有输入和输出流异步记录并压缩（Zstd）。
2. 在回放界面中能够像视频一样像素级重现当时的终端操作过程。
3. 敏感数据（如通过配置的正则表达式识别出的密码输入）能在日志中进行脱敏处理。
