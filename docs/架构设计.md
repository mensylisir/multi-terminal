# 架构设计

本文档定义了本多主机实时终端系统的宏观拓扑、核心通信链路以及数据的端到端流向。

## 1. 宏观系统架构图

整个系统被拆分为三大层次：前端浏览器呈现层、核心网关服务层（Gateway）、以及目标主机集群层。

```text
Browser（xterm.js + WebWorker）
  │
  │    ◄── 1. WebSocket（全双工，自定义二进制，多路复用） ──►
  │
Gateway（基于 Go 构建的高性能核心路由）
  ├── Session Manager (会话与状态管家)
  ├── SSH Manager (连接池与认证中心)
  ├── Multiexec Engine (多路输入分发核心)
  ├── Stream Router (聚合帧输出网关，20ms 定时聚合)
  ├── Diff Engine (ANSI Strip + Regex Filter + Hash 比对分析器)
  ├── Audit Service (异步、无阻塞的指令与流落盘服务)
  ├── Risk Control (实时高危指令嗅探与拦截)
  ├── Resize Controller (前端窗体与远端 PTY 的尺寸同步)
  ├── Flow Control (反压 Backpressure 限流引擎)
  ├── Context Checker (TUI/Shell 状态探测器)
  └── Resource Guard (全局系统句柄及 Goroutine 保护伞)
  │
  │    ◄── 2. 标准 SSH 协议 (加密，独立 PTY 通道) ──►
  │
SSH Hosts (受管物理机、虚拟机、容器实例)
```

## 2. 系统的核心本质与特征

该系统并非简单的 WebSSH 工具，而是一个复杂的**集中式管控平台**，其本质特性包含：

1. **多终端多状态系统**: 突破单页应用限制，在一个视窗内高度密集地管理数十个独立 PTY 及其生命周期。
2. **连接治理大坝**: 承担起前端海量 WebSocket 连接到后端异构 SSH 主机的阻抗匹配，做连接的汇聚与映射。
3. **高频指令多播 (Multicast)**: 作为代理人，安全且保序地将单点输入实时且同步地分发至广域网络上的各个节点。
4. **二进制信道复用引擎**: 将无数的零散字符输出流聚合，在一条 TCP 链路中打上 Tag 传输，解决 IO 瓶颈。
5. **动态阻尼流控 (Flow Control)**: 像大坝一样，既不过载前端浏览器，也不被远端日志洪流冲垮。
6. **无感旁路审计与风控**: 在数据双向奔流的同时，以旁路的方式完成高危拦截脱敏与全量录像。

## 3. 核心数据流说明

### 3.1 下行流 (输入广播)
`用户键盘交互` -> `主线程捕获` -> `Web Worker 封包 (二进制)` -> `WebSocket` -> `Gateway: Multiexec Engine` -> `根据 selectedSessions 切分流` -> `风控引擎拦截预检` -> `SSH Manager` -> `多台目标机 PTY`。

### 3.2 上行流 (输出聚合)
`多台机器并行输出` -> `Gateway SSH Manager 读取` -> `Flow Control(水位检查)` -> `Stream Router (暂存入缓冲池)` -> `(触发 20ms 定时器)` -> `打包成单帧 Frame` -> `WebSocket 发送` -> `Web Worker 解包拆流` -> `主线程 xterm.js 各自渲染更新`。
