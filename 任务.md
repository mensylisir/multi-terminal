# 开发任务拆分列表 (任务.md)

基于系统架构与设计文档，本平台按阶段拆解为以下开发任务，用于指导后续编码实现。

## 阶段一：基础设施与核心通信 (Phase 1)
此阶段构建系统骨架，打通 WebSocket 通信与基本的前后端交互能力。

- [ ] **任务 1.1: Gateway 项目初始化与配置管理**
  - 搭建 Go 项目脚手架，配置环境变数解析（端口、日志级别等）。
  - 实现基础的优雅停机（Graceful Shutdown）框架。
- [ ] **任务 1.2: WebSocket Server 基础实现**
  - 基于 Gorilla WebSocket 实现连接接管。
  - 定义并实现底层的心跳机制 (Ping/Pong)，超时自动断开。
- [ ] **任务 1.3: 二进制协议编解码器开发**
  - 定义 `FrameType`, `SessionCount`, `SessionId`, `Length`, `Data` 的序列化与反序列化逻辑。
  - 使用 `sync.Pool` 实现各种容量 Buffer 的复用池。
- [ ] **任务 1.4: 前端核心通信层与 Web Worker 初始化**
  - 创建 Vue/React 前端项目。
  - 编写 Web Worker 用于收发 WebSocket 二进制数据，并支持 DataView 解析与打包。
  - 实现主线程与 Worker 之间的双向消息机制。

## 阶段二：会话管理与终端接入 (Phase 2)
此阶段完成单机 SSH 连接与 xterm.js 界面呈现。

- [ ] **任务 2.1: Session Manager 模块**
  - 实现全局线程安全的 `sync.Map` 注册表，管理 Session 的创建、查询、销毁。
  - 定义 Session 状态机（active, detached, closed）。
- [ ] **任务 2.2: SSH Manager 模块**
  - 封装 `crypto/ssh`，实现客户端鉴权连接。
  - 申请 PTY 通道，暴露 Read/Write 接口供上层使用。
- [ ] **任务 2.3: 前端 xterm.js 渲染接入**
  - 集成 `xterm.js` 及其 WebGL 插件。
  - 将 Worker 解析出的流数据驱动给 xterm 实例，并将键盘输入回传给 Worker。
- [ ] **任务 2.4: 终端窗口自适应控制**
  - 监听前端浏览器 Window Resize 事件，计算并传递 Cols/Rows。
  - 后端接收 Resize 控制帧，调用 SSH `window-change`。

## 阶段三：聚合路由与流控缓冲 (Phase 3)
解决高并发下的小包碎片化和内存积压问题。

- [ ] **任务 3.1: 终端输出缓冲队列**
  - 为每个 Session 建立带上限（如 1MB）的 RingBuffer。
- [ ] **任务 3.2: Stream Router (20ms 聚合引擎)**
  - 实现定时 Ticker，每 20ms 收割所有活跃 Session 的缓冲输出。
  - 将收集的数据组装成一个聚合二进制帧下发。
- [ ] **任务 3.3: Backpressure 机制**
  - 监测 Buffer 水位，水位超 85% 时标记 Session 为慢节点并暂停从 SSH 侧 Read。
  - 水位回落后恢复 Read 流程。
- [ ] **任务 3.4: 前端乐观回显与流控**
  - 限制 xterm 的 Scrollback 上限为 5000 行。
  - 利用 `requestAnimationFrame` 对高频输出实施渲染节流。

## 阶段四：批量广播 (Multiexec) 与一致性保障 (Phase 4)
实现本系统的核心卖点：安全可控的多节点命令广播。

- [ ] **任务 4.1: 前端多终端布局与状态同步**
  - 引入 Zustand/Pinia 管理 activeSessionId 及 selectedSessionIds。
  - 支持快捷键开启/关闭 Multiexec 模式，多 xterm 同屏排版显示。
- [ ] **任务 4.2: Multiexec Engine 广播逻辑**
  - 拦截 active 终端输入，基于目标列表异步分发到对应 Session 的写入通道。
  - 保障输入顺序（FIFO），支持针对慢节点的针对性暂停与警告。
- [ ] **任务 4.3: 上下文状态探测 (Context Checker)**
  - 后端在广播前下发 `\x05` 探测，分析回显确定处于 `shell` 还是 `tui`。
  - 排除 `tui/unknown` 节点，向前端推送状态；前端呈现红色锁定遮罩。
- [ ] **任务 4.4: 终端回显锁定器**
  - 在前端实现 Multiexec 时，active 终端执行乐观回显。
  - 临时锁定所有被选终端的服务端回显窗口，稍后统一刷新解决重影问题。

## 阶段五：数据隔离审计与高可用 (Phase 5)
实现高阶特性：审计日志、差异比对及优雅升级。

- [ ] **任务 5.1: Diff Engine (输出差异识别)**
  - 实现 ANSI 颜色代码去除与 Regex 噪声过滤算法。
  - 对按行切分的输出进行 Hash 计算，辅助前端进行多机结果异同高亮。
- [ ] **任务 5.2: Audit Service (高保真审计)**
  - 实现命令脱敏与结构化日志落盘。
  - 集成 Zstd 库，对所有输入输出流进行异步实时压缩录像。
- [ ] **任务 5.3: 断线重连与状态恢复**
  - 前端 WS 异常断开触发后端 Session 转入 `detached`（不立即销毁 SSH）。
  - 支持前端带旧 SessionId 重新握手连接，拉取挂起缓冲数据恢复场景。
- [ ] **任务 5.4: 平滑升级 (Graceful Restart) 支持**
  - 拦截系统退出信号，阻断新连接进入。
  - 借助 Redis 持久化挂起 Session 元数据，完成进程间 fd 句柄接力。
